# Сценарий: стандартный платёж

**Участники:** ваш сервис (инициатор), {{PRODUCT_NAME}}, банк-партнёр.

**Предусловия:** клиент прошёл KYC; сумма в пределах дневного лимита
(иначе см. ветку ручного одобрения ниже).

## Шаги

1. `POST /v1/payments` — платёж создаётся, статус `processing`.
2. {{PRODUCT_NAME}} передаёт платёж банку-партнёру.
3. По колбэку банка статус становится `settled` (успех) или `rejected` (отказ).
4. Ваш сервис узнаёт итог через `GET /v1/payments/{id}` или вебхук.

## Исходы

| Статус | Что означает | Действие интегратора |
|---|---|---|
| `settled` | Расчёт завершён | Выдать товар/услугу |
| `rejected` | Отказ банка | Показать клиенту причину из ответа |
| `pending_approval` | Сумма выше лимита, ручная проверка | Ждать; нормальное время — до часа |

## Что считать нормой

Обычно платёж достигает финального статуса за секунды. Если статус
не меняется дольше часа — обратитесь в поддержку с `payment_id`.
