# Сценарии

Справочник API отвечает на вопрос «что делает эта ручка». Сценарий — на вопрос
«как из ручек собрать бизнес-результат»: принять платёж, провести возврат,
обработать отказ. Если ваши интеграторы пишут в саппорт «а в каком порядке
это вызывать?» — им не хватает именно этого раздела.

## Шаблон сценария

Каждый сценарий в этом разделе отвечает на семь вопросов:

1. **Кто участвует** — стороны и их роли.
2. **Предусловия** — что должно быть выполнено до старта.
3. **Шаги** — последовательность вызовов с условиями ветвления.
4. **Исходы** — все способы, которыми сценарий может закончиться, включая отказные.
5. **Частота и тайминги** — что считать нормой, а что зависанием.
6. **Почему правила такие** — лимиты, проверки и их происхождение.
7. **Границы ответственности** — где заканчивается ваша зона и начинается зона клиента.

Пример по шаблону: [Стандартный платёж](/scenarios/payment-standard).

:::caution Сценарии протухают
Спеку хотя бы линтит CI. Сценарии не проверяет никто: система меняется,
и через полгода текст уверенно врёт. Минимальная защита — ревью раздела
при каждом изменении статусной модели или лимитов (добавьте это в
чек-лист PR). Честное решение задачи «дока говорит правду» выходит за
рамки инфраструктуры — этому посвящена отдельная статья автора стартера.
